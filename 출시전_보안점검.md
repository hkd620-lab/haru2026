## 🔴 출시 전 필수 작업

### 1. Firebase App Check
**위험**: 앱 외부에서 Firebase에 직접 API 호출 가능
**해결**: Firebase 콘솔 → App Check → reCAPTCHA v3 또는 App Attest 활성화

```javascript
// firebase.ts에 추가 필요
import { initializeAppCheck, ReCaptchaV3Provider } from 'firebase/app-check';

initializeAppCheck(app, {
  provider: new ReCaptchaV3Provider('RECAPTCHA_SITE_KEY'),
  isTokenAutoRefreshEnabled: true,
});
```

---

### 2. Functions Rate Limiting (호출 횟수 제한)
**위험**: 1명이 반복 호출로 Gemini API 과금 폭탄 유발 가능
**해결**: Firestore에 호출 기록을 저장하여 분당/일당 제한

```javascript
// 예시: 하루 100회 제한
const today = new Date().toISOString().slice(0, 10);
const usageRef = db.doc(`users/${uid}/usage/${today}`);
const usage = await usageRef.get();
if ((usage.data()?.count ?? 0) >= 100) {
  throw new HttpsError('resource-exhausted', '일일 한도 초과');
}
await usageRef.set({ count: FieldValue.increment(1) }, { merge: true });
```

### 3. Functions 입력값 검증
**위험**: 악의적 데이터(비정상 크기, 타입) 전송 가능
**해결**: 각 Function에서 데이터 유효성 검사 추가

```javascript
// polishContent 예시
const { text, format } = request.data;
if (typeof text !== 'string' || text.length > 10000) {
  throw new HttpsError('invalid-argument', '잘못된 입력값');
}
if (!['일기','에세이','여행기록','텃밭일기','애완동물관찰일지','육아일기','선교보고','일반보고','업무일지'].includes(format)) {
  throw new HttpsError('invalid-argument', '유효하지 않은 형식');
}
```

### 4. 구독 검증 로직 (결제 연동 시)
**위험**: 프론트엔드에서 구독 상태를 판단하면 개발자 도구로 우회 가능
**해결**: 구독 여부는 반드시 서버(Functions)에서만 판단

```
❌ 프론트: if (user.isSubscribed) { 프리미엄 기능 허용 }
✅ 서버:  Functions에서 Firestore subscription 문서 확인 후 허용
```

결제 웹훅(Stripe 등) 처리 시 반드시 서명 검증 필요:
```javascript
const sig = req.headers['stripe-signature'];
stripe.webhooks.constructEvent(req.rawBody, sig, WEBHOOK_SECRET);
```

---

### 5. Gemini API 키 환경변수 등록
**위험**: API 키가 코드에 하드코딩되면 GitHub 노출 시 즉시 악용
**해결**: Firebase Functions 환경변수에만 저장

```bash
# 등록 방법
firebase functions:secrets:set GEMINI_API_KEY

# Functions 코드에서 사용
import { defineSecret } from 'firebase-functions/params';
const geminiKey = defineSecret('GEMINI_API_KEY');
```
