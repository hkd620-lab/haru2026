# HARU — 피그마 디자인 피드백 브리핑

> **앱 개요**: HARU는 감성 일기장이 아닌 **체계적인 기록 도구**입니다.
> 사용자가 날씨/기분/형식을 선택하고 글을 쓰면, AI(SAYU)가 내용을 정제해줍니다.
> 주요 사용자: 20–40대 한국어 사용자, 모바일 중심.

---

## 1. 디자인 시스템 현황

### 컬러 팔레트

| 역할 | 값 | 사용처 |
|---|---|---|
| Primary (Navy) | `#1A3C6E` | 버튼, 활성 아이콘, 브랜드 강조 |
| Background | `#FAF9F6` | 앱 전체 배경 (따뜻한 오프화이트) |
| Surface | `#FFFFFF` | TopBar, BottomNav, 카드 |
| Text Primary | `#333333` | 본문 |
| Text Secondary | `#666666` | 날짜, 부제 |
| Text Muted | `#999999` | 비활성 탭, 힌트 |
| Border | `#E5E5E5` | 구분선 |
| Destructive | `#D4183D` | 삭제/경고 |

### 레이아웃

- **TopBar**: 고정, 높이 56px — 뒤로가기 + 홈 버튼
- **BottomNav**: 고정, 높이 80px — 7개 탭 (아이콘 + 레이블)
- **콘텐츠 영역**: `max-width: 768px`, 좌우 패딩 16px
- **안전 여백**: 상단 56px, 하단 80px

## 2. 화면별 현황 설명

### 🏠 홈 (/)

- 로고 애니메이션(2초) 후 슬로건 등장
- 날짜 표시
- CTA 버튼: "오늘 기록 쓰기" (네이비 배경, 흰 텍스트, shadow-lg)
- **현재 문제**: 화면 중앙에 버튼 하나만 있어 여백이 너무 많음

### 📝 기록 작성 (/record)

- 날짜 자동 입력
- 기분 선택: `기쁨 / 평온 / 무미 / 울적 / 번잡` (5개 버튼)
- 날씨 선택: `쾌청 / 흐림 / 강우 / 굳음` (4개 버튼)
- 기온 선택: `폭염 / 온난 / 쾌적 / 쌀쌀 / 혹한` (5개 버튼)
- 형식 선택: `일기 / 에세이 / 선교보고 / 일반보고 / 업무일지 / 여행기록` (최대 3개)
- 저장 버튼 → 서재로 이동
- **현재 문제**: 옵션 버튼들의 선택/비선택 상태가 시각적으로 불명확

### 📚 서재 (/library)

- 날짜별 기록 목록
- 각 기록에 태그된 형식(일기, 에세이 등) 표시
- 형식 클릭 → 모달에서 내용 입력/편집
- **현재 문제**: 기록 카드 레이아웃이 단순하여 가독성 개선 필요

### ✨ SAYU (/sayu)

- AI가 정제한 기록 열람/편집
- 별점(1–5) 평가
- 원본 ↔ AI 정제본 토글 뷰
- **현재 문제**: AI 콘텐츠와 원본 콘텐츠의 구분이 시각적으로 약함

### 🔀 기록합침 (/merge)

- 여러 날의 기록을 합쳐 하나의 문서로 만들기
- **현재 문제**: 기능 자체가 직관적이지 않아 UX 가이드 필요

### 📊 통계 (/stats)

- 아직 미구현, Coming Soon 상태
- 월별/형식별/기분 트렌드 예정

### ⚙️ 설정 (/settings)

- 사용자 통계 요약 (총 기록 수, SAYU 생성 수 등)
- 데이터 내보내기 (JSON)
- 전체 삭제
- 로그아웃

## 3. 디자인 개선 요청 사항

### 우선순위 HIGH

1. **홈 화면 구성 개선**
   - 현재: 로고 + 날짜 + 버튼 1개만 존재, 과도한 여백
   - 요청: 최근 기록 미리보기 또는 동기부여 요소 추가 레이아웃 제안

2. **기록 작성 화면 — 옵션 선택 UI**
   - 현재: 선택/비선택 상태가 색상 변화만으로 구분
   - 요청: 선택 상태를 더 명확하게 표현하는 버튼/칩 디자인 (체크마크, 테두리 강화 등)

3. **BottomNav 탭 수 정리**
   - 현재: 7개 탭이 너무 많아 좁은 화면에서 레이블이 잘림
   - 요청: 탭 구조 개편 또는 계층화(More 메뉴 등) 방안 제안

### 우선순위 MEDIUM

4. **서재(Library) 카드 디자인**
   - 요청: 날짜 + 기분/날씨 이모지 + 형식 태그가 보기 좋은 카드 레이아웃

5. **SAYU 화면 — AI vs 원본 구분**
   - 요청: 두 모드가 명확히 구분되는 탭 또는 슬라이드 UI

6. **다크 모드 컬러 팔레트**
   - 현재 CSS에 `.dark` 클래스 있으나 미완성
   - 요청: Primary `#1A3C6E`의 다크 모드 대응 컬러 제안

### 우선순위 LOW

7. **온보딩/로그인 화면**
   - 현재: 기본적인 Google 로그인 버튼만 존재
   - 요청: 앱 가치 전달 + 로그인 유도 화면 디자인

8. **통계 화면 — 차트 디자인**
   - 요청: 구현 전 와이어프레임/목업 제안 (월별 캘린더 히트맵, 기분 도넛 차트 등)

## 4. Figma에 전달하는 방법

### 방법 A — Figma Tokens 플러그인 (디자인 토큰 임포트)
1. Figma에서 **Plugins → Figma Tokens** 설치
2. 플러그인 실행 → **Load from file** 선택
3. `figma/design-tokens.json` 파일 업로드
4. 컬러/타이포그래피/간격이 Figma 스타일로 자동 생성됨

### 방법 B — Figma Make (AI 디자인 생성)
1. Figma 웹에서 **Make** 기능 접근
2. 이 문서의 각 화면 설명을 프롬프트로 입력
3. 예시 프롬프트:
   ```
   Korean journal app "HARU". Mobile-first (375px wide).
   Home screen: centered logo animation, today's date, one CTA button "오늘 기록 쓰기".
   Primary color: #1A3C6E (navy). Background: #FAF9F6 (warm white).
   Clean, minimal, structured tool aesthetic — NOT a cute diary.
   ```

### 방법 C — 화면 캡처 첨부
1. 앱을 로컬에서 실행 (`npm run dev`)
2. 각 화면 캡처 → Figma 파일에 참조 이미지로 배치
3. 위 개선 요청 사항을 Figma 코멘트로 달아 디자이너와 협업
