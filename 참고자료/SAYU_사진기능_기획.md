# SAYU 대표 사진 기능 기획서

## 📅 작성일
2026년 2월 25일

## 🎯 기능 개요
각 형식별로 다듬어진 SAYU와 함께 대표 사진 1장을 첨부할 수 있는 기능

## 📋 최종 결정사항

### 1. 업로드 시점
- ✅ **SAYU 다듬기 후** 사진 업로드
- 사용자 작성 → AI 다듬기 → SAYU 모달 → [선택] 사진 업로드 → 저장

### 2. 이미지 사양
- **최대 크기**: 500KB 이하로 자동 압축
- **권장 해상도**: 800x600px
- **형식**: JPG (용량 효율적)
- **필수 여부**: ❌ 선택 사항 (필수 아님)

### 3. 저장 구조

#### Firebase Storage 경로
```
users/
  └─ {userId}/
      └─ sayu_photos/
          └─ {recordId}_{formatKey}.jpg
```

**예시:**
```
users/abc123/sayu_photos/2026-02-25_diary.jpg
users/abc123/sayu_photos/2026-02-25_essay.jpg
```

#### Firestore 데이터 구조
```typescript
{
  // 기존 필드
  diary_sayu: "AI가 다듬은 글...",
  
  // 추가 필드
  diary_sayu_photo: "https://firebasestorage.../abc123.jpg"
}
```

## 👍 장점

1. **시각적 매력**
   - 글만 있는 것보다 훨씬 풍부한 경험
   
2. **기억 강화**
   - 사진이 그날의 순간을 더 생생하게 만듦
   
3. **감정 전달**
   - 글로 표현하기 어려운 순간을 사진으로 보완
   
4. **완성도 향상**
   - 일기장처럼 느껴지는 완성도
   
5. **차별화**
   - 다른 일기 앱과의 차별점 확보

## 👎 단점 및 대응 방안

### 1. 비용 증가
**문제점:**
- Firebase Storage 사용료 발생
- 사용자 증가 시 비용 급증 가능

**대응:**
- 썸네일 크기로 압축 (500KB 이하)
- 무료 한도 내 운영 가능 (월 5GB)

### 2. 속도 저하
**문제점:**
- 이미지 업로드/로딩 시간 추가

**대응:**
- 자동 압축으로 용량 최소화
- 로딩 인디케이터 표시

### 3. 개발 복잡도
**문제점:**
- 이미지 업로드 UI 개발
- 압축/최적화 기능 필요

**대응:**
- 단계별 구현 (MVP 먼저)
- 라이브러리 활용 (browser-image-compression)

## 🔒 보안 고려사항

### Firebase Storage Rules
```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /users/{userId}/sayu_photos/{fileName} {
      allow read: if request.auth != null;
      allow write: if request.auth != null 
                   && request.auth.uid == userId
                   && request.resource.size < 2 * 1024 * 1024; // 2MB 제한
    }
  }
}
```
