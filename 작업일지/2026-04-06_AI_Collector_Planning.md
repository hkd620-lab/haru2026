# [2026-04-06] AI 지식 수집기 & HARU2026 연동 통합 기획안

## 1. 프로젝트 개요
*   **목표**: 외부 AI(ChatGPT, Gemini, Claude 등)의 대화 내용을 버튼 하나로 수집하여 개인 DB(Fire)와 HARU2026 앱에 동시 저장하는 시스템 구축.
*   **핵심 가치**: 사용자 편의성(자동화) + 지식의 주권(로컬 소장 가능).

## 2. 주요 아키텍처 구상
### A. 두 개의 우체통 (Dual Server Sync)
*   **대상**: 창고용 Firestore (`my-ai-library-74805`) + 하루 앱용 Firestore (`haru2026-8abb8`).
*   **방식**: 크롬 확장 프로그램에서 두 프로젝트의 API Key를 모두 사용하여 각기 다른 컬렉션에 동시에 `Fetch` 요청을 보냄.

### B. 사용자 저장소 선택권 (Privacy & Ownership)
*   **서버 저장**: 편리한 동기화와 앱 확인용.
*   **로컬 저장**: 마크다운(.md) 파일로 사용자 하드디스크에 즉시 다운로드.
*   **개인 클라우드**: 사용자의 개인 구글 드라이브 폴더에 저장 (사용자 서버 비용 절감 및 데이터 소유권 강화).

### C. 수집 대상 및 방식
*   **외부 AI**: 크롬 확장 프로그램을 통해 DOM에서 텍스트 추출 (Selectors로 관리).
*   **내부 AI (AG)**: 앱 내부 API 연동을 통해 확장 프로그램 없이 직접 하루 앱으로 전송.

## 3. 세부 기획 사항
*   **구독자 경험**: 복잡한 설치 단계 축소 (웹스토어 정식 등록 혹은 간편 북마클릿 제공).
*   **데이터 스키마**: `title`, `content`, `source`, `category`, `uid`, `timestamp` 등의 규격을 하루 앱의 '작업일지' 형식에 최적화하여 맞춤.
*   **비즈니스 모델**: 로컬 저장은 기본 무료 기능, 클라우드 동기화(앱 연동)는 유료 구독 모델로 확장 가능성 제안.
