### 2단계: 권한 문제 발견
**문제:**
```
Permission 'iam.serviceAccounts.signBlob' denied
code: 'auth/insufficient-permission'
```

**원인:** 
- customToken 생성에 필요한 권한이 없음
- Firebase Admin SDK가 토큰에 서명할 수 없음

**해결:**
1. Google Cloud Console → IAM → 서비스 계정
2. `398138267500-compute@developer.gserviceaccount.com` 선택
3. **"Service Account Token Creator"** 역할 추가
4. 5분 대기 (권한 전파 시간)

**교훈:** Gen 2는 보안이 강화되어 명시적 권한 설정 필요!

### 3단계: 네이버 photoURL 문제
**문제:**
```
The photoURL field must be a valid URL
code: 'auth/invalid-photo-url'
```

**시도한 것들:**
```typescript
// 시도 1: null 체크
const photoURL = naverUser.profile_image || null;
// ❌ 실패

// 시도 2: 빈 문자열 체크
let photoURL = null;
if (naverUser.profile_image && naverUser.profile_image.trim() !== '') {
  photoURL = naverUser.profile_image;
}
// ❌ 여전히 실패
```

**최종 해결:**
```typescript
// photoURL 완전히 제거
await admin.auth().createUser({ uid, email, displayName });
// photoURL 파라미터 자체를 안 넣음
```

**교훈:** 빈 문자열도 invalid URL로 간주됨. 아예 제거가 정답!

### 4단계: 이메일 충돌 문제
**문제:**
```
auth/email-already-exists

```

**원인:** 
- 이전 테스트로 생성된 계정들
- Google 로그인 계정과 이메일 충돌

**해결:**
- Firebase Authentication에서 테스트 계정 삭제
- 깔끔한 상태에서 재테스트

## 배운 교훈들

### 🔥 Firebase Gen 2는 Gen 1과 완전히 다름
- 환경변수: .env 자동 로드 안 됨
- 권한: 명시적으로 설정해야 함
- 보안: 훨씬 강화됨

### 🔍 에러 로그 = 보물지도
- firebase functions:log 필수
- 단계별로 어디서 멈추는지 확인
- 에러 메시지를 정확히 읽기

### 🚀 작동 우선, 보안은 나중
1. 일단 코드에 박아서 작동시키기
2. 작동 확인 후 Secret Manager로 이전
3. 완벽보다 진행이 중요

### 💡 환경변수 = 배포의 가장 어려운 부분
- 로컬에서는 잘 됨
- 배포하면 안 됨
- 가장 많은 시간 소요
